# 全谱系博弈训练系统 v3 — AI 对手决策引擎设计

> 配套 `engine/` 模块。目标：让 6 个对手具备**差异化决策 + 跨回合记忆 + 情绪状态机**，形成"性格→行为→破绽"的可博弈体验。

---

## 1. 设计理念

v2.0 的对手 = `Math.random() < 阈值`，无记忆、无情绪、彼此雷同。v3 把每个对手建模为：

```
对手 = 性格参数(静态) + 情绪状态(动态) + 跨回合记忆(对玩家的认知) + 决策策略(性格专属函数)
```

决策 = `f(场景上下文, 性格, 当前情绪, 对玩家的记忆)` —— 加权规则 + 状态机，**不引入机器学习**，保证可解释、可复盘、零依赖。

---

## 2. 统一接口

```js
// engine/opponent-ai.js
function decide(opponentId, scenarioKey, ctx) {
  // ctx: { round, history, playerLastAction, scenarioState }
  const persona  = OPPONENTS[opponentId];          // data/opponents.js 静态参数
  const memory   = Memory.get(opponentId);          // 对玩家的累计认知
  const mood     = Mood.get(opponentId);            // 当前情绪状态
  const strategy = STRATEGY_MAP[opponentId];        // 性格专属策略函数
  const decision = strategy(ctx, persona, memory, mood);  // → { move, reason }
  Mood.update(opponentId, ctx, decision);           // 情绪演化
  Memory.observe(opponentId, ctx.playerLastAction); // 更新对玩家认知
  return { ...decision, mood: Mood.get(opponentId) };
}
```

返回的 `reason` 与 `mood` 通过 `ai:decided` 事件传给场景与复盘层。

---

## 3. 跨回合记忆（engine/memory.js）

对每个对手，维护一份"对当前玩家的画像"：

```js
memory = {
  rounds: 0,
  playerCoopCount: 0,        // 玩家合作次数
  playerDefectCount: 0,      // 玩家背叛次数
  playerCoopRate: 0.5,       // 滑动合作率
  avgConcession: 0,          // 玩家平均让步幅度（谈判类）
  aggressionIndex: 0.5,      // 玩家激进度（极端出价频率）
  betrayalStreak: 0,         // 当前连续背叛
  trustReturned: 0.5,        // 玩家在信任博弈中的返还率
  promiseKept: 0.5,          // 玩家履约度
}
```

- `observe(action)`：每回合后更新上述指标（用指数滑动平均，近期行为权重更高）。
- 记忆**仅在单局内有效**（每局开始重置）；可选项：跨局长期记忆（Phase 4 评估，默认关闭以免难度失控）。

## 4. 情绪状态机（engine/mood.js）

每个对手维护内部情绪，取值 0~1：

```js
mood = {
  trust: 0.5,       // 对玩家的信任
  anger: 0.0,       // 愤怒（被背叛/被压迫累积）
  patience: 1.0,    // 耐心（随回合与僵局递减）
  confidence: 0.5,  // 谈判自信（受比分/让步影响）
}
```

**演化规则示例**：
- 玩家合作 → `trust += 0.1`，`anger -= 0.05`
- 玩家背叛 → `trust -= 0.15`，`anger += 0.2`（强硬型 `anger` 增益更大）
- 玩家极端施压 → `patience -= 0.2`，`confidence -= 0.1`
- 僵局/拖延 → `patience -= 0.1`

情绪反过来调节决策（如 `anger > 0.6` 触发报复，`patience < 0.2` 触发摊牌/离场）。

## 5. 六对手差异化策略

> 每个 `strategy(ctx, persona, memory, mood)` 返回 `{ move, reason }`。下面给出各对手的**决策核心**与**可被识破的破绽**。

### 5.1 陈逻辑 · 理性分析型（rational）
- **核心**：纯收益最大化 + 镜像。囚徒困境用 **Tit-for-Tat**（首轮合作，之后复制玩家上轮）；谈判按期望值计算让步，记忆中玩家 `aggressionIndex` 高则减小让步。
- **记忆运用**：强。严格按玩家历史合作率调整。
- **情绪**：弱（`anger` 影响小），但 `trust` 影响合作意愿。
- **破绽**：对情感诉求免疫，但**完全可预测**——玩家用稳定合作可建立稳定互惠；用数据/客观标准（费希尔）可高效说服。

### 5.2 林敏感 · 情感驱动型（emotional）
- **核心**：决策被 `mood` 主导。`trust` 高 → 慷慨；`anger` 高 → 非理性报复（甚至损人不利己）。
- **记忆运用**：中。更看"最近一次"玩家是否伤害了它（近期权重极高）。
- **情绪**：极强。波动大。
- **破绽**：易被"贴标签/战术性同理心"安抚（沃斯），也易被激怒。玩家可在其 `mood.trust` 高位时推进协议。

### 5.3 钢铁王 · 强硬鹰派型（aggressive）
- **核心**：高开价、小让步、施压。谈判让步幅度最小；囚徒困境高背叛倾向。
- **记忆运用**：中。若玩家展现强 BATNA / 合理反压（`aggressionIndex` 适中且坚定），`confidence` 下降 → 妥协。
- **情绪**：`anger` 增益大；但 `patience` 低，僵局久了会松动。
- **破绽**：**合理反压时容易妥协**（PRD 设定）。玩家硬刚 + 亮替代方案可压其 `confidence`；一味退让则被吃干抹净。

### 5.4 和谐李 · 合作共赢型（cooperative）
- **核心**：高合作、主动让步、寻求双赢。囚徒困境高合作；谈判让步幅度大。
- **记忆运用**：强但善意——但**会学习**：玩家若反复利用其善意（高 `aggressionIndex` + 低合作率），`betrayalStreak` 触发后转为"宽容版 TfT"（报复但易复合）。
- **情绪**：`trust` 易升，`anger` 升得慢。
- **破绽**：过度善意易被利用；但 v3 给它"被骗 N 次后觉醒"的反制，避免成为人形提款机。

### 5.5 影子张 · 操纵控制型（manipulative）
- **核心**：心理战。隐藏意图、制造紧迫感、报价反复。信任博弈中"钓鱼"（先高返还诱导玩家加大投入，再背刺）。
- **记忆运用**：高。专挑玩家弱点（玩家 `trustReturned` 高 → 加大背刺）。
- **情绪**：伪装——对外 `mood` 与真实意图不一致。
- **破绽**：**被识破后失效**。v3 设"识破机制"：玩家若拒绝紧迫感、要求书面确认、识破其反复（用心理防御战术），触发 `exposed=true`，此后其操纵收益归零、被迫转常规策略。

### 5.6 稳健吴 · 风险规避型（risk-averse）
- **核心**：偏好确定性，回避极端。囚徒困境倾向稳定合作（怕被双背叛的坏结果）；谈判接受"足够好"而非"最优"。
- **记忆运用**：中。玩家制造不确定性/风险叙事会使其让步。
- **情绪**：`patience` 较高，但 `confidence` 易受"损失框架"打击。
- **破绽**：**压力下过度让步**（PRD 设定）。玩家用损失厌恶框架（"不成交您将失去…"）+ 提供"安全的次优选项"可高效收割。

## 6. 决策权重模板（可调）

各对手策略内部用统一的加权打分挑选 move：

```
score(move) = w1·收益期望
            + w2·情绪倾向(mood)
            + w3·记忆驱动(对玩家的认知)
            + w4·性格基线(persona)
            + 噪声(小幅随机，保留不可完全预测性)
```

不同对手的 `w1..w4` 权重不同（如理性型 w1 最高，情感型 w2 最高，操纵型 w3 最高）。具体权重在各 `strategies/*.js` 内定义并附注释，便于调参与复盘解释。

---

## 7. 与复盘的联动

`decide` 返回的 `reason`（如"因你连续2次背叛，信任降至0.2，转入报复"）与 `mood` 快照，被 `round:end` 收集，供 `analytics/replay.js`：
- 绘制对手情绪曲线
- 在复盘中解释"对手为何这样应对"
- 据此给玩家针对性改进建议
