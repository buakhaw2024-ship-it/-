## 🏗️ 架构速览

```
src/
├── core/       ① 事件总线 + 全局状态 + 启动
├── data/       ② 静态数据（对手/场景/宗师/策略）
├── engine/     ③ AI 决策引擎（6对手差异化 + 跨回合记忆 + 情绪状态机）
├── scenarios/  ④ 7 大博弈场景（统一接口 + 注册表）
├── ui/         ⑤ 视图层（只渲染，不含业务逻辑）
└── analytics/  ⑥ 心理画像 + 复盘评分 + 持久化

build.js  →  dist/game_trainer.html （单文件，可离线分发）
```

数据流：`UI 操作 → emit 事件 → engine/scenario 处理 → 更新 store → emit 状态变更 → UI 重渲染`

## ▶️ 运行方式

- **开发态**：本地起 http server（ES Module 需 http 协议），打开 `src/index.html`。
  ```bash
  cd src && python -m http.server 8000   # 然后浏览器访问 http://localhost:8000
  ```
- **分发态**：`node build.js` → 双击 `dist/game_trainer.html` 即可（无需服务器）。

## ✅ v3 正式版功能全览（Phase 4-6）

### Phase 4 — 复盘与画像深化
| 功能 | 描述 |
|------|------|
| 逐回合评分 | 每轮决策 0-100 分，颜色分级（绿/黄/红）|
| 转折点标注 | 自动定位关键失误或最佳一手 |
| 对手情绪曲线 | 内联 SVG 展示信任/愤怒随回合演化 |
| 改进建议 | 表现分析 + 针对特定对手的专项技巧 |
| 真实心理画像 | 8 维指标基于玩家真实决策序列计算 |
| 识破对手 | 每局 2 次消耗式"peek"，查看对手当前情绪状态 |

### Phase 5 — 难度系统 + 隐藏 Boss
| 功能 | 描述 |
|------|------|
| 四档难度 | 初级 / 中级 / 高级 / 终局挑战，**真正影响博弈机制**（对手慷慨度/强硬度/接受阈值）|
| 难度信息分级 | 初级显示弱点，中级显示行为信号，高级封锁信息 |
| 隐藏 Boss | 「极限交易型 / 高压锚定型」对手，**仅宗师级 + 终局挑战难度解锁** |
| Boss 自适应 | 跨回合记忆：你越无条件让步它越索取，亮 BATNA / 反压才略松动 |
| Boss 专项复盘 | 5 点高压锚定应对诊断（抽象谈判策略，非政治评论）|

### Phase 6 — 看板与短板分析
| 功能 | 描述 |
|------|------|
| 场景胜率 | 7 个场景各自场次 + 胜率分级 |
| 对手胜率 | 6 基础对手 + Boss 战绩 |
| 短板分析 | 基于真实画像的 6 类短板检测（看板 + 心理档案双展示）|
| 难度分布 | 按难度统计训练场次 |

### Phase 7 — 可视化「谈判桌」UI 改造
| 功能 | 描述 |
|------|------|
| 对手头像 | 每个对手专属 emoji 头像（理性🧮/情感🌊/鹰派🦅/合作🤝/操纵🎭/稳健🛡️/Boss👑），圆形光环呼吸动画，Boss 金色光晕 |
| 关系温度条 | 红→黄→绿渐变轨道，滑块随每次 AI 出招（`ai:decided` 事件）平滑滑动，实时反映信任 − 愤怒 |
| 对话气泡 | 玩家选择（右·青）/ 对手应答含策略推理（左·暗）/ 系统事件（中·黄），取代扁平日志 |
| 回合时间线 | 空心=待开始、青色脉冲=当前回合、暗灰实点=已完成；阶段制场景显示阶段进度 |
| 结果屏头像 | 复盘页带回对手头像与类型，贯穿整局视觉身份 |

> 改造全部基于纯 CSS + HTML 字符串生成；引擎仍不碰 DOM，UI 仅通过事件总线通信，架构约束保持不变。
