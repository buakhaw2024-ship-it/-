# 话术生成修复说明（全谱系谈判博弈模拟系统）

## 问题现象
- **前端直连模式**：话术生成长时间「✨ 正在据对手这句话生成贴合话术…」延迟、最终无显示。
- **后端安全模式**：选项卡片直接显示「⚠️ 生成失败 · 点上方「🔄 重新生成话术」重试」。
- 结果：开启 AI 增强后，回应选项的话术不可用，游戏无法有效进行。

## 根因
两种模式都把「大模型可达」当成了**硬依赖**，失败时没有回退：

1. **后端安全模式**：默认后端地址是 `http://localhost:8787/api/ai/negotiation-turn`。
   单文件 HTML 本地打开时并没有这个服务，`fetch` 立即被拒绝（连接被拒），
   于是卡片直接落到失败分支，显示「生成失败」。

2. **前端直连模式**：浏览器从 `file://` / `blob:null` 源**直接**请求大模型 API。
   跨域（CORS / `Origin: null`）会让请求挂起或被拒；而代码里**只有一个 12 秒的外层超时**，
   请求未返回前，卡片原本写好的话术被「正在生成…」占位符覆盖，于是「延迟 + 无显示」，
   12 秒后再翻成「生成失败」。

关键缺陷：游戏其实**已经内置了一套完整的本地规则话术生成器**（`_aiLocalRewrite`），
教练复盘面板失败时也已经会回退到「本地诊断（大模型不可用）」——
但**回应选项卡片没有做同样的回退**。

## 修复（`index.html` 内联脚本，`_aiOptFlow` / `_aiDynBeat`）
让选项话术与教练面板一样**优雅降级**：

1. **先用本地话术立即铺底**：进入回合即用 `_aiLocalRewrite`（命中不到时退回选项原话术）
   同步填好每张卡片——永远不空白、不卡「正在生成」。
2. **大模型只作为「升级」**：未开启 AI 时直接用本地话术；开启后在后台请求，
   成功则把卡片升级为大模型话术。
3. **任何失败 / 超时都保留本地话术**，不再显示「生成失败」。
4. 外层超时从 **12s 收紧到 9s**，直连挂起时更快回退。

净效果：三种模式都能正常运行——
- 关闭 AI：本地话术（一直可用）；
- 后端模式无服务：瞬间回退本地话术；
- 直连模式 CORS/挂起：先显示本地话术，能连通时再升级为大模型话术。

## 使用建议
- **本地离线试玩**：保持 AI 增强关闭即可，话术全部由本地规则生成。
- **前端直连**：设置「直连平台」新增 **DeepSeek V4 Flash / V4 Pro** 预设，已默认走
  **OpenRouter**（跨域友好，浏览器可直连），所以 API Key 请填 **OpenRouter** 的；模型 id
  为 `deepseek/deepseek-v4-flash` / `-pro`（若 OpenRouter 命名不同，在 Model 字段改即可）。
  若你的 DeepSeek 官方 Key 允许浏览器跨域，把 Base URL 改回 `https://api.deepseek.com/v1`、
  Model 去掉 `deepseek/` 前缀即可。API Key 存本地浏览器 `localStorage`，仅适合本机自用。
- **后端安全**：部署 `server/` 代理后，在设置「后端接口」输入框填入地址并点「保存后端地址」
  （此前只显示、无法修改，现已可编辑）。公开部署请在地址末尾加 `?k=<访问令牌>`。
  未部署/不可达时不报错，自动回退本地话术。

## 验证
- 完整内联脚本（约 20MB）通过 `node --check` 语法校验。
- 对 `_aiOptFlow` 做了运行时行为测试：AI 关闭→本地话术；AI 失败→保留本地话术（不出现 `ai-fail`/「生成失败」）；AI 成功→升级为大模型话术。
- 重建后的 `index.html` 结构完好（7 个打包资源、模板可解码、加载器完整）。
