# 传奇剧情链 · 对话语境匹配修复说明

## 问题现象

在「传奇剧情链」对局中，**对手的发问**与**玩家所选策略选项下的应对话术**上下文不匹配——
例如对手（帘后）问「这铁路，洋人掺了多少股？」，而玩家可选的「查敌情 / 定边界 / 借外压 / 给台阶」
四句话术却仍在讲「折子停在军机处还是帘内」，答非所问。接入大模型后，对手台词被随机生成，
错配现象更明显。

## 根因

`renderStoryPlay()`（剧情链对局屏）里：

- **对手台词**取自 `sAct.beats[roundInAct]`——每一幕有 **4 个节拍**，逐回合推进、内容各不相同。
- **玩家选项话术**取自 `sAct.voices[key]`——每一幕**只有一套**，4 个回合**始终不变**。

于是节拍随回合升级，而玩家话术固定不动；这套话术通常是针对该幕"整体主题/最后一拍"写的，
导致与当前回合实际显示的对手节拍对不上。接入大模型时，对手台词变成模型生成的随机内容，
而玩家选项依旧是静态脚本，两者来自**互相独立**的上下文，错配进一步放大。

## 修复方案

核心原则：**对手"当前节拍"与五个策略选项的玩家话术，必须取自同一回合上下文。**

### 1. 按"回合/节拍"解析话术（本地剧本）

新增 `resolveStoryVoices(sAct, script, roundInAct)`，解析优先级：

1. 节拍对象自带 `beat.voices`
2. `act.beatVoices[round]`（**新增数据结构**：每个节拍一套话术）
3. `act.voices`（向后兼容，旧数据原样工作）
4. 链级 `script.voices`

并新增 `storyBeatText()` 统一取节拍文本（兼容字符串/对象两种节拍写法）。

`renderStoryPlay()` 改为：对手台词与五个选项话术都经由上述解析器，按**当前回合**取值；
上一手回显也按"当时回合"解析，保证回显与彼时语境一致（`StoryChain.step` 包装后会记录
`shownVoice`）。

> 已为 `cixi_story_chain` 第一幕补全四个节拍的 `beatVoices`（即截图所示场景），
> 其余幕/链可按同一结构增量补写；未补写处自动回退到 `act.voices`，行为不变。

### 2. 接入大模型：同上下文一次性共生成（`StoryDialogueAI`）

新增独立模块 `StoryDialogueAI`，开启 AI 增强后，对**当前回合**发起**一次**模型调用，
要求模型在同一上下文内**同时**返回：

```json
{
  "opponentLine": "对手此刻说的一句话",
  "voices": {
    "info": "查敌情 / 探询信息",
    "reframe": "定边界 / 重定议题",
    "pressure": "借外压 / 施压",
    "build": "给台阶 / 建立信任",
    "lock": "锁条款 / 拍板"
  }
}
```

由于对手发问与五句回应来自**同一次调用、同一上下文**，模型被显式约束"五句必须直接回应
同一个 opponentLine"，从根本上消除随机错配。

- 结果按 `chainId:actIndex:roundInAct` 缓存于 `state.aiStoryRounds`，回合内只生成一次。
- 复用既有 `NegotiationLLMBridge` 的直连 / 后端两种模式与 API Key 配置。
- 生成中显示「AI 正在生成匹配台词…」，成功显示「AI · 语境已匹配」，失败显示
  「AI 暂不可用 · 已回退剧本台词」并退回本地剧本（带 `beatVoices` 的解析）。
- 校验不足（有效话术少于 3 句）时**宁可回退剧本**，不展示半成品错配内容。

### 保持的边界

与既有 AI 增强层一致：**仅增强台词，不参与分数 / 胜负 / 经验 / 解锁 / 存档结算**。
全部改动向后兼容，AI 关闭且未补 `beatVoices` 时行为与原版一致。

## 改动位置（运行时模板，即打包文件第 181 行内联源码）

- `STORY_SCRIPTS.cixi_story_chain.acts[0]`：新增 `beatVoices`（4 拍 × 5 策略）。
- `renderStoryPlay()`：对手节拍 + 选项话术 + 回显，全部改为按回合解析，并接入 AI 缓存。
- `renderStoryPlay.afterRender`：进入回合时按需触发 `StoryDialogueAI.ensureRound()`。
- 新增 `storyBeatText` / `resolveStoryVoices` / `buildStoryRoundContext` / `StoryDialogueAI`，
  并包装 `StoryChain.step` 以记录 `shownVoice`。

## 如何扩展到其它幕 / 剧情链

给对应 `act` 增加 `beatVoices` 数组即可（顺序与 `beats` 对齐，每项含
`info/reframe/pressure/build/lock` 五键）。缺省项自动回退 `act.voices`。
接入大模型后，所有剧情链的每个回合都会动态生成语境匹配的台词，无需逐句手写。
