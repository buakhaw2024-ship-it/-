# 传奇试炼场 22 位人物场景图片设计接入文档 v1.0

## 1. 概述

本文档定义了「全谱系谈判博弈模拟系统 v5.12」传奇试炼场模块中 22 位传奇人物的电影化场景图与肖像图的设计规范、技术接入方式和生成工作流。

### 1.1 目标

- 为 22 位传奇导师各提供 1 张电影化场景图（scene）和 1 张半身肖像图（portrait）
- 替换现有 CSS 渐变 fallback 标题卡，提升视觉沉浸感
- 保持与现有代码架构的无缝兼容，支持渐进式接入

### 1.2 人物总览

| 分类 | 数量 | 稀有度 |
|------|------|--------|
| UR · 传奇导师 | 16 位 | UR_LEGEND |
| SSR · 理论宗师 | 6 位 | SSR_MASTER |
| **合计** | **22 位** | - |

## 2. 图片设计规范

### 2.1 场景图（Scene）

| 属性 | 规格 |
|------|------|
| 尺寸 | 1200 × 520 px |
| 格式 | WebP（首选）/ PNG |
| 风格 | 电影化写实，暗调光影 |
| 安全区 | 左侧 38% 为文字叠加区，保持暗色/留空 |
| 人物位置 | 右侧 60% 区域 |
| 禁忌 | 无文字、无水印、无 Logo、无 UI 元素 |

**安全区说明：**

```
┌──────────────────────────────────────────┐
│ 安全区（暗色渐变）  │   人物 + 场景      │
│  ← 38% →           │   ← 62% →          │
│                     │                     │
│ [场景标题药丸]      │   [人物主体]        │
│ [色调标签]          │   [场景细节]        │
└──────────────────────────────────────────┘
```

### 2.2 肖像图（Portrait）

| 属性 | 规格 |
|------|------|
| 尺寸 | 400 × 400 px |
| 格式 | WebP（首选）/ PNG |
| 风格 | 半身肖像，与场景图同一视觉语言 |
| 裁切 | 适合圆形裁切（头部居中偏上） |
| 背景 | 暗色，可有模糊场景元素 |

### 2.3 通用质量要求

- 无变形手指/额外手指
- 面部清晰、高分辨率
- 无卡通/动漫/Q版风格
- 无赛博朋克头盔（除非场景指定）
- 无夸张奇幻铠甲
- 历史人物参考真实形象但做艺术化处理

## 4. 技术接入架构

### 4.1 数据层

现有 `LEGEND_SCENE_IMAGES` 对象为核心接入点：

```javascript
const LEGEND_SCENE_IMAGES = {
  mentor_sunzi: {
    scene: '',      // ← 注入场景图路径
    portrait: '',   // ← 注入肖像图路径
    sceneTitle: '吴国伐楚前夜 · 军帐庙算',
    visualTone: '青黑、铜绿、军帐烛火、竹简地图',
    safeArea: 'left'
  },
  // ... 其余 21 位
};
```

### 4.2 渲染层

现有函数调用链：

```
renderLegendSceneVisual(mentorId)
  → getLegendSceneImage(mentorId)    → scene 路径 或 ''
  → getLegendSceneMeta(mentorId)     → {sceneTitle, visualTone, safeArea}
  → 有图 → background-image + gradient overlay
  → 无图 → .legend-scene-fallback CSS 渐变标题卡

renderLegendPortraitVisual(mentorId)
  → getLegendPortraitImage(mentorId) → portrait 路径 或 ''
  → 有图 → <img class="legend-portrait-img">
  → 无图 → .legend-portrait-fallback 文字占位
```

### 4.3 补丁注入流程

```
legend_scene_image_patch.js
  │
  ├─ 定义 SCENE_IMAGE_REGISTRY（22 个路径映射）
  │
  ├─ validateAndInject()
  │   └─ 遍历 registry，对 status==='ready' 的条目
  │      注入 scene/portrait 路径到 LEGEND_SCENE_IMAGES
  │
  ├─ setupSceneLoadingStates()
  │   └─ MutationObserver 监听 .legend-scene-visual 新增节点
  │      → 预加载背景图 → 添加 loaded/error CSS 类
  │
  └─ preloadReadyAssets()
      └─ 批量预热已就绪图片到浏览器缓存
```

## 5. AI 图片生成工作流

### 5.1 提示词来源

`LEGEND_IMAGE_PROMPTS` 对象包含 22 位人物的 `scenePrompt` 和 `portraitPrompt`。

`LEGEND_NEGATIVE_PROMPT` 包含统一的负向提示词。

### 5.2 生成步骤

1. 从 `LEGEND_IMAGE_PROMPTS[mentorId]` 取出 `scenePrompt`
2. 附加 `LEGEND_NEGATIVE_PROMPT`
3. 生成 1200×520 场景图
4. 从同一条目取出 `portraitPrompt`
5. 附加 `LEGEND_NEGATIVE_PROMPT`
6. 生成 400×400 肖像图
7. 保存为 WebP 格式到对应目录
8. 在 `legend_scene_image_manifest.json` 中将该人物 `status` 改为 `"ready"`

### 5.3 质量检查清单

- [ ] 左侧 38% 安全区无人物遮挡
- [ ] 面部清晰可辨
- [ ] 无变形/多余手指
- [ ] 无可读文字/水印/Logo
- [ ] 色调与 `visualTone` 描述一致
- [ ] 肖像图适合圆形裁切
- [ ] 文件大小 < 300KB（WebP）

## 6. 回退机制

当图片未就位时（`scene` 和 `portrait` 为空字符串），系统自动使用现有 fallback：

- **场景图 fallback**：`.legend-scene-fallback` — 每位人物独立的 CSS 渐变背景 + 场景标题药丸
- **肖像图 fallback**：`.legend-portrait-fallback` — 人物名字前两字 + 金色渐变背景
- **MENTOR_PORTRAITS fallback**：v5.8 引入的 22 张做旧 SVG 肖像（如存在）

回退层级：`portrait` → `scene` → `MENTOR_PORTRAITS` → 文字占位
